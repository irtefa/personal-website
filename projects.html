Sherlog

October 5, 2013 - October 9, 2013

I and Stephen Lee initially started this project as an assignment for a Distributed Systems class. Things changed when I casually decided to put it up on Hacker News. Within a few hours, it was in the top ten most popular posts that day. None of my other projects is as popular as Sherlog (76 stars and counting!). I do have to confess though that the purpose of this project was not to propose something novel, rather it was intended to experiment with Go and its famous go routines.

Sherlog is a distributed logging system that allows you to run system grep calls in all the machines in your system and returns the relevant matches back to the user. Grep is executed on each machine separately. The machines are independent in the sense that they do not wait for each other to complete the execution of Grep. The client receives the response as soon as a machine completes executing grep on its log. As the machines are not waiting for each other there is no blocking in the execution process.

Fitquo

December, 2012

We created a social network for aspiring fitness enthusiasts, Fitquo. Fitquo was selected as one of the best projects out of 52 other projects in the class. It provides a common domain for fitness trainers and for people who want to get in shape.

We used Tornado as our webserver, MySQL as our backend database and Redis as a client side cache.

Sentipede

November 3, 2012

A web-application to detect your mood over time and other parameters based on your Facebook data. Created in 24-hours for UIUC Facebook Hackathon 2012. The primary goal for this project was to learn basic sentiment analysis methods. We also wanted to be fluent in facebook's graph API. Frankly, this is not my proudest projects. As it was built in a very short period of time (less than 24 hours), most of the code is very hacky. Nevertheless, I, along with Suharsh Sivakumar, Koushik Roy and Rishi Bajekal learned a lot about basic sentiment analysis and facebook's graph API from this project.

We used Tornado as our webserver, MongoDB as our backend database and D3 as our client side JavaScript framework. You can find the code here.
