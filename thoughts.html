  1. Limitations of JavaScript

    March 24, 2013

    Recently, I answered a question on Quora about the limitations of JavaScript. The language itself is extremely powerful and reasonably fast. I thought I would share my answer here.

    JavaScript is a very powerful language that runs on your web browser. The fact that it runs on your 'web browser', creates a lot of limitations. Even if there are multiple ways to work around these problems, they are largely based on extended libraries or sending server side calls. JavaScript, inherently does not have solutions to these problems.

    Does not have multithreading: JavaScript does not have multithreading capabilities.

    Cannot access web pages hosted on a different domain: You cannot access files that belong to a different domain name using solely JavaScript. Its based on the Same origin policy. You can workaround this using HTML5's new feature "postMessage" interface or using server side scripts using AJAX calls.

    Cannot access databases: JavaScript cannot access backend databases without a server side script. However, there are certain client side databases that you can access via JavaScript e.g. Local Storage.

    Cannot write to files on the server: Just like the aforementioned limitation, you cannot directly write to a file on the server side without actually sending an AJAX request that runs a server side script to do that for you. However, its also a good thing at the same time that you cannot write your files to the server as it raises an additional security concern.

    Depends a lot on the browser: JavaScript depends a lot on the browser it runs on. Different browsers interpret JavaScript differently as the browsers have diverse JavaScript engines. Its always safer to test your code in all major browsers before pushing it to production.

    Inability to use local devices: You cannot use most of your local devices with JavaScript.

    Option to disable JavaScript: A lot of people tend to disable JavaScript in their browsers due to various security reasons. Think about all the websites that are highly dependent on client side scripts. If the user disables JavaScript, the web app is broken and it can't run on the browser which not only creates a problem for the web app but also for the user who is trying to enjoy a service!
